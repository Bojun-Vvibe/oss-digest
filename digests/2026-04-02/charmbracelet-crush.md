# charmbracelet/crush — 2026-04-02

The headline is **v0.55.0** ([release](https://github.com/charmbracelet/crush/releases/tag/v0.55.0)), which lands two substantial features merged the same day.

The most consequential change is the **server/client architecture refactor** ([#2455](https://github.com/charmbracelet/crush/pull/2455), commit `28f2087`). Crush is now split into a long-running HTTP server that manages multiple workspace sessions and a lightweight client that talks to it via REST. It is gated behind `CRUSH_CLIENT_SERVER=1`, which spawns a detached server process before showing the TUI if one isn't already running. This is explicitly framed as "big and important" by the author (aymanbagabas) and supersedes the earlier draft [#1110](https://github.com/charmbracelet/crush/pull/1110), which was closed in favor of this cleaner take. Anyone integrating Crush programmatically or planning remote/multi-session usage should read this PR carefully — it reshapes the runtime model.

The second headline feature is **built-in skills** via `go:embed` ([#2466](https://github.com/charmbracelet/crush/pull/2466), commit `2832d52`). Built-ins live under a `/crush` path prefix (the original `crush://` scheme was dropped because LLMs mistook it for MCP URIs), and user-level skills with the same name override them. The first shipped built-in is **`crush-config`**, letting users ask Crush to edit its own `crush.json` instead of hand-editing. A new config option also allows disabling skills, including built-ins — useful for users who want to keep the tool surface minimal.

Rounding out the day: a small modernization pass ([#2548](https://github.com/charmbracelet/crush/pull/2548), commit `e3df84e`) bringing the codebase in line with `task modernize`, plus the routine `v0.55.0` version bump (`3befaaa`) and an auto-update commit (`7572ce1`).
