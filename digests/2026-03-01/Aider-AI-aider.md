## New "between" edit format proposed (#4875)

[PR #4875](https://github.com/Aider-AI/aider/pull/4875) by `muravvv` introduces a new `between` edit format, positioned as an alternative to the existing `diff` / `udiff` formats for LLMs that struggle to emit valid SEARCH-REPLACE blocks. Today, such models effectively had only the `whole` format available, which is expensive in tokens and brittle on larger files.

The author's motivation, per the PR description, is the observation that most LLMs in normal chat mode can not only produce a code snippet but also describe *where* it should be pasted (e.g. "after function X" or "before statement Y"). The `between` format is designed to capture exactly that pattern: the model identifies anchor context surrounding the change rather than reproducing the original block verbatim, which is what trips weaker models up on SEARCH-REPLACE.
