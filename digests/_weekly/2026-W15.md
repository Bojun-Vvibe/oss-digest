## Trend summary

The biggest story this week is **Codex flipping its Realtime experience to WebRTC by default** and reshaping how voice sessions hand off to the coding agent. [openai/codex#17188] makes the v2 WebRTC path the default, [openai/codex#17183] defaults new realtime sessions to `gpt-realtime-1.5`, and [openai/codex#17278] renames the realtime delegation tool to `background_agent`. Around that, [openai/codex#17264] streams background-agent progress mid-turn, [openai/codex#17306] queues `response.create` while a response is open, [openai/codex#17519] budgets thread context to 300 tokens per turn for realtime startup, and [openai/codex#17520] mirrors typed user text into the realtime stream. Voice is no longer a side experiment — it now shares the same tool surface and context discipline as the TUI.

**Token discipline was a cross-repo theme.** Crush shipped a flag-gated rewrite of every tool description ([charmbracelet/crush#2592]) — about 98% smaller for non-Bash tools, an estimated ~120K tokens saved per session — and added two self-introspection tools, `crush_info` and `crush_logs`, so the agent can debug its own config and logs. Codex removed the in-statusline context meter ([openai/codex#17420], [openai/codex#17170]) after persistent negative feedback that it read like a quota gauge.

**MCP server churn continues, mostly in two directions.** The official `modelcontextprotocol/servers` repo spent the week closing dozens of "add my server to the README" PRs ([modelcontextprotocol/servers#3902], [#3884], [#3886], [#3888], [#3909], [#3916], [#3923] and many more) — the maintainers are routing submissions to the new MCP Server Registry instead. Meanwhile real bug fixes landed: a fetch-server fallback when Readability strips SSR content ([modelcontextprotocol/servers#3879], [#3894]), and a Windows UNC-path normalization fix in the filesystem server ([modelcontextprotocol/servers#3920]). On the consumer side, Codex shipped MCP Apps part 3 with direct tool-call support ([openai/codex#17364]), expanded tool search to custom MCPs ([openai/codex#16944]), paused MCP elicitation timers from counting against tool-call time ([openai/codex#17566]), and auto-approved MCP elicitations in Full Access mode ([openai/codex#17164]). opencode added configurable OAuth redirect URIs for remote MCP servers in v1.4.3.

**Model routing and provider plumbing.** LiteLLM's headline merge is the Advisor orchestration loop for non-Anthropic providers ([BerriAI/litellm#25579]), letting non-native models participate in advisor flows; alongside it, [BerriAI/litellm#25409] adds Wandb's Kimi-K2.5 and MiniMax-M2.5, [BerriAI/litellm#25287] expands Azure container routing, and [BerriAI/litellm#25527] cuts Prometheus latency-bucket cardinality. Crush v0.55.1 quietly fixed a class of local-model failures ([charmbracelet/crush#2554]): LM Studio rejected `max_tokens:0` and unset context windows triggered immediate auto-summarize on brand-new sessions. opencode v1.4.3 added "fast mode" variants for Claude and GPT, and Codex added a `prolite` plan type ([openai/codex#17419]).

**OpenHands ran a sustained V1 API migration sprint.** Settings ([OpenHands/OpenHands#13759]), secrets ([#13770]), health ([#13724]), git installations/repositories with pagination ([#13790]), models search ([#13799]), and `/users/me` ([#13778], [#13822]) all moved under `/api/v1`, with the legacy V0 endpoints marked deprecated and the frontend cut over in [#13803], [#13811], [#13865], [#13870], [#13871], [#13875], [#13877], [#13887]. Microagent management UI was deleted outright ([#13830], [#13835]). Two notable security fixes: WebSocket auth moved to a first-message frame to keep `sk-oh-*` tokens out of proxy access logs ([#13777]), and MCP URL secrets are now redacted from logs ([#13834], [#13838]).

**TUI and agent-loop polish.** opencode shipped a steady stream of TUI fixes (subagent click targets, Ctrl+Z undo on Windows, hidden scrollbar restored) and a sweeping internal refactor by @kitlangton converting tools to `Tool.defineEffect` and tearing down facade wrappers across ~40 commits. Codex added Ctrl+R reverse history search ([openai/codex#17550]) and a configurable thread-title statusline item ([openai/codex#17187]). Cline shipped v3.78.0 with a dedicated Spend Limit Reached UI ([cline/cline#10207]). Aider was quiet — one feature commit enabling overeager mode for Claude Sonnet 4.5.
