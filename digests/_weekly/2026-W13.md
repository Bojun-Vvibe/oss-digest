The week's headline belongs to **Cline 3.76.0**, which makes the new **Cline Kanban** experience the default surface for the CLI and ships a launch modal inside the VSCode webview to nudge existing users over [cline/cline#9914, cline/cline#9963, cline/cline#9988]. The migration is not subtle — Saoud's commits add a Kanban description in place of the old `--tui` hint, fall back through package managers to spawn it, and even add a Windows shell-mode fix when Kanban couldn't launch on win32. Combined with **repeated tool-call loop detection** [cline/cline#9933] and **chunked `read_file` with `start_line`/`end_line`** [cline/cline#9711], this release is the clearest signal yet that Cline is repositioning from "VS Code extension that runs an agent" toward "agent runtime with multiple frontends."

**Codex's quiet refactor is enormous.** bolinfest carved the tool layer apart into a new `codex-tools` crate across at least six PRs — tool spec models, configured specs, code-mode adapters, local-host, collaboration, and utility specs [openai/codex#16047, #16129, #16132, #16138, #16141, #16154]. In parallel etraut-openai **deleted the legacy TUI split** [openai/codex#15922], renamed `tui_app_server` to `tui` [openai/codex#16104], and **removed voice transcription** [openai/codex#16114] and remaining custom-prompt support [openai/codex#16115]. The MCP layer also got an increased startup timeout [openai/codex#16080] and a bypass for read-only tool checks [openai/codex#16044].

**MCP server churn** continued in `modelcontextprotocol/servers`: the merge queue is mostly stalled (only 2 of 17 PRs landed, both quite old), but the open queue is dominated by ghost-authored fixes to filesystem (UNC paths, Windows EPERM on rename) [modelcontextprotocol/servers#3633, #3632], git root validation [modelcontextprotocol/servers#3631], and memory-server schema validation [modelcontextprotocol/servers#3627, #3634], alongside the usual wave of third-party server submissions.

**TUI polish** showed up across the board: opencode 1.3.5/1.3.6 fixed token double-counting after the AI SDK v6 upgrade [anomalyco/opencode#19758] and a broken variant-dialog filter [anomalyco/opencode#19917]; Crush got variable-height prompt input [charmbracelet/crush#2468], smarter file-completion ranking [charmbracelet/crush#2433], and a diff-view fix for line-ending normalization [charmbracelet/crush#2499]; OpenHands shipped an old/diff/new toggle in the file-changes viewer [OpenHands/OpenHands#13519]. Crush also tightened safety by dropping the view/fetch tool max read from 5 MB to 1 MB [charmbracelet/crush#2447] and exporting `CRUSH=1`/`AGENT=crush` env vars so subprocesses can detect they're inside an agent [charmbracelet/crush#2484].
