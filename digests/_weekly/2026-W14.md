The biggest story this week is the rise of **"skills" as a first-class primitive across multiple agents**. Crush shipped [charmbracelet/crush#2466], introducing built-in skills (with a `crush-config` skill bundled), per-skill disabling, and skill metadata surfaced in the session UI [charmbracelet/crush#2541] — landing in [v0.55.0](https://github.com/charmbracelet/crush/releases/tag/v0.55.0). Codex moved in the same direction, sorting skill mentions by display name in the TUI [openai/codex#16710] and instrumenting subagent analytics [openai/codex#15915]. OpenHands is wrestling with the rougher edges of the same idea: a cluster of PRs aims to surface skill-loader failures upstream rather than swallow them [OpenHands/OpenHands#13782, OpenHands/OpenHands#13783], and a paginated skills cursor bug got fixed in [OpenHands/OpenHands#13743]. Skills/subagents are clearly the new battleground for "how does the agent know what it can do."

If you embed OpenHands, this is the week to start your migration.
