The headline today is a session-corruption fix: [#2597](https://github.com/charmbracelet/crush/pull/2597) (merged, commit `efe42e8`) hardens the agent against malformed multimodal tool results. Bad base64 in image payloads could previously brick sessions; the fix validates base64, swaps invalid data for a placeholder, and also prevents double-encoding so models actually receive valid images. It closes the long-standing #1782 and is worth reading if you've seen mysterious session breakage involving images or multimodal tools.

The other notable merge is [#2592](https://github.com/charmbracelet/crush/pull/2592), which slashes tool-call description sizes by roughly 98% for every tool except Bash. The maintainers estimate ~120K tokens saved per session. The change is gated behind `CRUSH_SHORT_TOOL_DESCRIPTIONS=1` while they verify there's no regression in tool-selection quality before flipping it on by default. Anyone running long sessions or tracking token spend should try the flag and report back.

On the new-PR side, [#2605](https://github.com/charmbracelet/crush/pull/2605) proposes an `additional_dirs` config option so the agent can read/list directories outside the working tree without permission prompts, plus a complementary `restrict_to_project` option that hard-blocks any tool access outside the allowed set. Paths support `~` and env-var expansion, and symlinks are resolved to block path-traversal tricks. It targets the common multi-repo workflow pain point and fixes #922 and #937 — likely to attract review attention given the security surface.

Also still open from earlier in the cycle but updated today: [#2418](https://github.com/charmbracelet/crush/pull/2418) adds a full Go profiling package (`internal/profiling`) covering CPU, memory, trace, block, and mutex profiling, wired into the root command via persistent flags. Useful infrastructure for anyone chasing performance regressions, though it's a larger surface and has been in flight since mid-March.

No releases, no new issues, and only a CLA-signing commit (`e9ee98f`) beyond the two merges above.
