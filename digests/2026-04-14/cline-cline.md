The most consequential change today is a deadlock fix in terminal command handling: `ask("command_output", ...)` could remain pending after the terminal process completed or errored, leaving Cline stuck. PR [#10269](https://github.com/cline/cline/pull/10269) adds an explicit release path during terminal lifecycle transitions (`completed`, `error`, timeout) and wires a targeted callback from `Task` to resolve the pending ask state, plus regression tests. If you've seen Cline hang after a shell command finishes (issue #10235), this is the fix to read.

Two prompt-correctness fixes for non-native tool-calling variants also landed. PR [#10200](https://github.com/cline/cline/pull/10200) adds the missing `use_subagents` tool to the hardcoded `TOOL_USE_SECTION` override templates for GLM-5, Hermes, and XS — these variants bypass auto-generated tool descriptions via `.overrideComponent()`, so the tool was registered but invisible to the model. A follow-up still-open PR [#10270](https://github.com/cline/cline/pull/10270) generalizes the lesson: it adds `use_skill` docs to the GLM and Hermes overrides and introduces a regression guard in `integration.test.ts` that fails if hardcoded `TOOL_USE` variants omit configured tools, to prevent future drift. Worth watching if you maintain prompt templates.

On the storage side, PR [#10264](https://github.com/cline/cline/pull/10264) adds Azure Blob Storage support with a custom implementation (Azure isn't S3-API-compatible), expanding remote storage options beyond S3.

Two Aikido security autofix PRs ([#10231](https://github.com/cline/cline/pull/10231), [#10232](https://github.com/cline/cline/pull/10232)) targeting critical axios RCE/SSRF, basic-ftp, and node-forge CVEs were **closed without merging** — the maintainers determined the affected code paths aren't actually used (basic-ftp is an unused transitive dep; node-forge crypto methods aren't invoked). Worth noting if you're tracking supply-chain posture.

Still open and potentially impactful: PR [#10087](https://github.com/cline/cline/pull/10087) proposes adding an optional `stream?: boolean` flag to `ApiHandlerOptions` to fix HTTP 400 errors from vLLM and other inference servers that don't support streaming.
