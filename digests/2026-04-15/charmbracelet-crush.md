The most ambitious open PR is **#2606 ("feat: split-pane tree, tab manager, and cross-platform PTY")** by ahostbr, still in review. It introduces three foundational packages — `internal/split/` (binary split tree for pane layout), `internal/tabmgr/` (tab manager with per-tab split trees, focus cycling, JSON persistence), and `internal/pty/` (cross-platform PTY abstraction using ConPTY via direct syscall on Windows, no cgo). This would be the groundwork for terminal multiplexing inside Crush, with each pane potentially hosting its own session, shell, or agent. If you only read one diff today, read this one — it's a sizable architectural addition that will shape how Crush handles concurrent agent work.
