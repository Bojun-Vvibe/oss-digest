The most consequential item is issue [#2649](https://github.com/charmbracelet/crush/issues/2649), a routing bug worth reading if you run custom providers. With `options.disable_provider_auto_update: true`, when `crush.json` declares a custom model whose `id` collides with the short form of an embedded catalog entry's dated id, crush sends the **embedded dated id** on the wire instead of the user-declared `id`. Truly novel ids work fine, so the issue is specifically about catalog-id collisions overriding user config. The reporter notes this broke routing against an OpenAI/Anthropic-compatible proxy. No fix is linked yet.

On the PR side, [#2643](https://github.com/charmbracelet/crush/pull/2643) (open, by `pi128`) targets two reasoning/thinking-content bugs in the TUI. First, real-time reasoning from Claude, DeepSeek, etc. was invisible during streaming because `RawRender()`'s cache was keyed only by width — the first render cached empty content and subsequent `SetMessage()` updates hit the stale cache. The fix bypasses the cache while `isSpinning()` is true. The PR also implements a previously missing toggle handler for showing/hiding reasoning. Worth a review pass if you care about streaming UX; not yet merged.
