The other new PR is **#3982** (fetch server), fixing a real interop issue with LibreChat and other clients that send explicit `null` for optional Pydantic parameters. The Fetch model typed `max_length`, `start_index`, and `raw` as `int`/`bool`, so Pydantic rejected `null` before defaults applied. The fix adds a `mode="before"` field validator that maps `None` to each field's declared default while preserving the existing `gt=0`, `lt=1_000_000`, `ge=0` constraints. Closes #2035.
