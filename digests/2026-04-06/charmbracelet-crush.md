The headline today is **v0.55.1** ([release](https://github.com/charmbracelet/crush/releases/tag/v0.55.1)), a patch focused on local-model compatibility. It ships two fixes from contributor @Kartik33 in #2554: Crush no longer sends `max_tokens: 0` when `MaxOutputTokens` is unset (LM Studio rejected this), and it skips auto-summarization when `context_window` is 0, which previously caused brand-new sessions against local models (LM Studio, Ollama, llama.cpp) to immediately reset with a "previous session was interrupted" message. If you run local models through Crush, upgrade.

Two notable new open PRs landed for review:

- **#2568** (`feat: support Bedrock cross-region inference model IDs and annotate 1M models`) picks up the leftover crush-side work from the abandoned #2321. It teaches Crush to accept `us.anthropic.*`, `eu.anthropic.*`, and `ap.anthropic.*` cross-region inference IDs, and surfaces the 1M context window for Opus 4.6 / Sonnet 4.6 (data already shipped in catwalk v0.33.2 via charmbracelet/catwalk#214) in the model picker. Worth a look if you're on Bedrock.
- **#2563** (`feat: add undo/redo support for session messages and file restoration`, closes #1300) is a more ambitious change: a non-destructive undo that sets a `revert_message_id` marker (hiding rather than deleting later messages until the next prompt, enabling redo), plus file restoration using the existing `history` table snapshots taken before every `edit`/`write`/`multiedit`. Agent-created files are deleted from disk on undo. Multi-step undo is supported. Design is worth scrutinizing given the destructive blast radius.

Also merged earlier but closed today: **#2522** conditionally hides image-related keybindings (`ctrl+f`, file picker) when the active model doesn't support image attachments — a small but visible UX cleanup.

No new issues or notable bug reports in the window. Commit activity is limited to the release tag, the #2554 merge, and a CLA bot signoff.
