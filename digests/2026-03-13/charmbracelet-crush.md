If you customize Crush's appearance or run it over a translucent terminal, this is the change to look at.

The other user-visible fix is Bedrock auth ergonomics: previously Crush accepted Bedrock credentials only via `aws login` or the `AWS_BEARER_TOKEN_BEDROCK` env var, and pasting the key directly into the input failed validation. PR #2407 lifts that restriction so users can simply paste the bearer token — worth flagging for anyone who's been confused by the auth flow on AWS Bedrock.

On the dependency side, the OpenAI SDK has been bumped to v3 (#2406), paired with a corresponding change in the upstream `charmbracelet/fantasy` repo (PR #168 there). The diff itself is a small chore commit (`451faa7`), but anyone tracking provider behavior or pinning versions downstream should be aware.

Internally, two refactors landed that don't change behavior but reshape how the app is wired:

- A larger config refactor (#2395, merged just inside the window) introduces a `ConfigStore` and `Scope` abstraction, making `config.Config` immutable and routing access through helpers that understand global vs. workspace scope. The stated motivation is to stop passing mutable config around and to lay groundwork for per-workspace overrides — useful context if you're about to touch config code or planning workspace-scoped features.
- Commit `e44211d` migrates the `run` and `login` commands off direct app access and onto the client API, continuing an apparent trend toward a cleaner internal API boundary.
