Today the loader calls `getattr(litellm, var)` without a default, so any litellm release that renames or removes an exception class raises an uncaught `AttributeError` and breaks aider entirely. The fix switches to `getattr(litellm, var, None)` and `continue`s past missing attributes. The PR wraps the cache write so a non-writable cache degrades gracefully rather than blocking startup. Useful for anyone running aider in hardened or containerized setups.
