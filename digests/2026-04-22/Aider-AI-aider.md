The day's headline is a proposed fix for a prompt injection vulnerability in Architect mode ([#5065](https://github.com/Aider-AI/aider/pull/5065), still open, draft=false). The PR targets the multi-agent orchestration flow where the Architect's proposal was forwarded to the Editor via `editor_coder.run(with_message=content, preproc=False)`. Because `preproc=False` skipped standard input processing, hostile instructions embedded in untrusted repo files (e.g., a poisoned `README.md`) could be relayed verbatim to the Editor and acted on, enabling attacker-controlled edits to land as commits. The PR removes that bypass so Editor input goes through normal preprocessing.
