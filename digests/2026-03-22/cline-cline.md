The root cause is globby's `gitignore: true` option: under the hood, globby eagerly walks the **entire** directory tree up front to discover every `.gitignore` file before doing any pattern matching. On large repos (or repos with deep `node_modules`, build artifacts, etc.), that pre-scan can exhaust memory and crash the extension before `listFiles` ever returns its (bounded) result set.

The fix replaces globby's built-in gitignore handling with a more targeted approach so that ignore rules are applied during traversal rather than via an upfront full-tree scan. This is the kind of bug that tends to bite power users with monorepos or large workspaces — exactly the population most likely to hit context-gathering paths repeatedly — so it's worth pulling if you've seen unexplained extension crashes when opening big projects.
