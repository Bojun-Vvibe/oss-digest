## Repo-map symlink fix proposed (#4891)

Contributor `yshch` opened [#4891](https://github.com/Aider-AI/aider/pull/4891) (still open, not draft) to stop the repo map from treating symlinks-that-point-to-directories as if they were regular files. In repos containing such symlinks, users currently see spurious warnings like:

> Repo-map can't include /path/to/repo/path/to/directory
> Has it been deleted from the file system but not from git?

The root cause described in the PR: git tracks symlinks as blob objects (storing the target path as the blob's content), so they show up in git's file enumeration alongside real files. When Python resolves the path on disk, it follows the link to a directory, and the repo-map code then fails to treat it as a file. The fix excludes directory-symlinks from the repo-map file listing.

This is a small, targeted bug fix rather than a behavior change — relevant primarily to anyone running aider in repos that intentionally check in symlinks to directories (monorepos, vendored trees, dotfiles-style setups). Worth reviewing if you maintain the repo-map code path; otherwise low-impact for most users.
